## 一、架构设计

### 1.1 Cesium Viewer 单例模式

整个应用共享**唯一**一个 Cesium Viewer 实例，由 `App.vue` 挂载时创建，`Pinia Store` 全局持有引用。

**为什么单例？** Cesium Viewer 创建开销极大（WebGL 上下文初始化、Worker 线程池、纹理/Shader 编译），多实例会耗尽 GPU 内存。所有功能模块共享同一场景，通过 `entities` 和 `dataSources` 隔离各自的可视化内容。

```
App.vue (mounted)
  └─ useCesium(containerRef)
       └─ new Cesium.Viewer(container, opts)
            └─ viewerStore.setViewer(viewer)  ← 存入 Pinia
                 └─ 所有视图通过 useViewer() 获取
```

**关键代码** (`src/composables/useCesium.js`)：

```js
// Cesium Ion Token（用于访问 Cesium 官方影像/地形服务）
Cesium.Ion.defaultAccessToken = CESIUM_TOKEN

// 创建 Viewer，关闭不需要的 UI 控件以减少性能开销
const viewer = new Cesium.Viewer(container, {
  timeline: false,           // 时间轴 — 不需要
  animation: false,          // 动画控件 — 不需要
  baseLayerPicker: false,    // 底图选择器 — 用自己的
  geocoder: false,           // 地理编码搜索 — 不需要
  infoBox: false,            // 信息弹窗 — 不需要
  selectionIndicator: false, // 选中指示器 — 不需要
  sceneMode: Cesium.SceneMode.SCENE3D,
})

// 关键场景参数
viewer.scene.globe.enableLighting = true            // 开启光照（地形阴影）
viewer.scene.globe.depthTestAgainstTerrain = true   // 深度检测（Entity 可被地形遮挡）
viewer.scene.globe.maximumScreenSpaceError = 2      // 地形精度（越小越精细，越耗性能）
```

**销毁清理** (`onUnmounted`)：遍历 `_trackedDataSources` 移除所有数据源 → 销毁 Viewer（释放 WebGL 资源）→ `$reset()`  Store。

## 二、坐标系处理 — GCJ-02 ↔ WGS-84

### 2.1 为什么需要坐标转换

- **Cesium 使用 WGS-84**（全球通用大地坐标系）
- **高德地图 API 使用 GCJ-02**（中国国家测绘局加密坐标系，俗称"火星坐标系"）
- 在中国境内，GCJ-02 与 WGS-84 存在 **100~700 米** 的系统性偏移
- 如果底图用高德/腾讯（GCJ-02），路况数据也需保持 GCJ-02 才能对齐

### 2.2 转换算法

实现完整的 GCJ-02 ↔ WGS-84 双向转换（`src/composables/useCoordinate.js`）：

```js
class CoordinateConverter {
  static #PI  = Math.PI
  static #A   = 6378245.0       // 克拉索夫斯基椭球长半轴
  static #EE  = 0.00669342162296594323  // 第一偏心率平方

  // 纬度偏转量
  static #transformLat(x, y) { /* 多项式拟合 */ }

  // 经度偏转量
  static #transformLng(x, y) { /* 多项式拟合 */ }

  // 中国境外检测（境外无偏转）
  static #outOfChina(lng, lat) {
    return lng < 72.004 || lng > 137.8347 || lat < 0.8293 || lat > 55.8271
  }

  // WGS-84 → GCJ-02（正向，供 API 请求前转换）
  static wgs84ToGcj02(lng, lat) {
    if (this.#outOfChina(lng, lat)) return [lng, lat]  // 境外不转
    // ... 偏转量计算 ...
    return [newLng, newLat]
  }

  // GCJ-02 → WGS-84（逆向迭代相减法，供 API 响应后转回）
  static gcj02ToWgs84(lng, lat) {
    if (this.#outOfChina(lng, lat)) return [lng, lat]
    // 计算偏转量 → 减去偏转 → 输出 WGS-84
    return [lng * 2 - newLng, lat * 2 - newLat]
  }
}
```

### 2.3 自动适配策略

`TrafficService` 会根据当前底图**自动决定是否做 GCJ-02 → WGS-84 逆向转换**：

```js
// 检测当前影像图层是否为 GCJ-02 底图
function isGcj02Imagery() {
  for (const layer of viewer.imageryLayers) {
    const url = layer.imageryProvider?.url || ''
    if (url.includes('autonavi') || url.includes('gtimg')) return true
  }
  return false
}

// GCJ-02 底图：keepGcj02 = true，跳过转回 WGS-84（路况线与底图对齐）
// Cesium 底图：keepGcj02 = false，转回 WGS-84（路况线位置正确）
const keepGcj02 = isGcj02Imagery()
```

## 三、交通态势 — 矩形拆分与并发查询

### 3.1 高德 API 限制

高德交通态势 API 要求矩形对角线 **≤ 10 km**，超出返回错误。

### 3.2 自适应拆分算法

```js
#splitBBox(bbox) {
  const queue = [bbox]
  const result = []
  const maxDepth = 2  // 最多拆 2 层 → 最多 4 块

  while (queue.length && result.length < (1 << maxDepth)) {
    const [w, s, e, n] = queue.shift()
    const d = this.#diagonal(w, s, e, n)

    if (d <= 10_000) {
      result.push([w, s, e, n])  // 已满足限制，直接入结果
      continue
    }

    // 沿较长边二等分
    if (Math.abs(e - w) >= Math.abs(n - s)) {
      const mid = (w + e) / 2
      queue.push([w, s, mid, n], [mid, s, e, n])   // 左右分
    } else {
      const mid = (s + n) / 2
      queue.push([w, s, e, mid], [w, mid, e, n])   // 上下分
    }
  }
  return result
}
```

### 3.3 并发控制

子矩形每 4 个一组并发请求（`Promise.allSettled`），失败不阻塞其他：

```js
for (let i = 0; i < bboxes.length; i += 4) {
  const batch = bboxes.slice(i, i + 4)
  const batchResults = await Promise.allSettled(
    batch.map(b => this.#requestSingle(b, opts))
  )
  // 只收集 fulfilled 的结果
}
```

### 3.4 去重

多个子矩形可能覆盖同一条道路，按 `road.id` 去重：

```js
const seen = new Set()
const deduped = results.filter(r => {
  if (seen.has(r.id)) return false
  seen.add(r.id)
  return true
})
```

## 四、视线分析 — 最大斜率 LOS 算法

### 4.1 算法原理

与 ArcGIS / QGIS / GRASS 的 Viewshed / LOS 工具一致。

**核心思想**：从观察者出发，沿目标方向逐采样点比较"该点对观察者的斜率"与"此前所有点的最大斜率"。

```
slope_j = (terrainH[j] - observerH) / distance[j]

若 slope_j > maxSlope_sofar  →  该点可见，更新 maxSlope
若 slope_j ≤ maxSlope_sofar  →  该点被前方更高地形遮挡
```

### 4.2 关键实现

```js
function analyzeMaxSlope(sampledCartos, distances, observerH, runId) {
  const visibilityMask = [true]  // 观察者自身总是可见
  let maxSlope = -Infinity

  for (let j = 1; j < sampledCartos.length; j++) {
    const terrainH = sampledCartos[j].height
    const dist = distances[j]
    if (dist <= 0) { visibilityMask[j] = true; continue }

    const slope = (terrainH - observerH) / dist

    if (slope > maxSlope) {
      visibilityMask[j] = true
      maxSlope = slope         // ← 更新最大斜率阈值
    }
    // else: 被遮挡 → visibilityMask 保持 false
  }

  // 目标点特殊处理：用目标实际高度（含离地 AGL）重新判定
  if (!visibilityMask[last]) {
    const tgtSlope = (tgtHeight - observerH) / totalDist
    if (tgtSlope > maxSlope) visibilityMask[last] = true
  }

  return { visibilityMask, vp: (visibleCount / N * 100).toFixed(1) }
}
```

### 4.3 海拔高度说明

- **椭球高（Ellipsoid Height）**：Cesium 内部使用的高度，即相对于 WGS-84 椭球面的高度
- **地形高（Terrain Height）**：`sampleTerrainMostDetailed` 返回的地表高度，也是椭球高
- **AGL（Above Ground Level）**：离地高度，观察者眼高 = 地形高 + AGL（如人眼高 1.7m）
- **绝对高度**：`obsHeight = terrainH + observerAGL`

## 五、可视域 — 批量地形采样优化

### 5.1 核心优化

原始方案每条射线逐点 `globe.pick`（360 × 10 = 3600 次同步调用），改为：

```
生成所有采样点经纬度（360 × 10 = 3600 个 Cartographic）
        ↓
  sampleTerrainMostDetailed（一次异步调用，内部并行采样）
        ↓
 逐射线判定 LOS（纯数学计算，无 GPU 交互）
```

性能提升：**~10×**（3600 次 GPU 拾取 → 1 次批量地形查询）

### 5.2 关键代码

```js
// 阶段 1：生成 360 条射线 × 每线 10 步 = 3600 个采样点
const allPositions = []   // Cartographic[]
const rayMeta = []        // 每条射线的 { startIdx, count, angle }

for (let i = 0; i < 360; i++) {
  const angle = (i / 360) * Math.PI * 2
  rayMeta.push({ startIdx: allPositions.length, count: 10, angle })
  for (let s = 1; s <= 10; s++) {
    const dist = (s / 10) * radius
    const target = geodesicTargetFast(ellipsoid, obsCarto, angle, dist)
    allPositions.push(new Cartographic(target.lon, target.lat, 0))
  }
}

// 阶段 2：批量采样（一次异步调用）
const sampled = await Cesium.sampleTerrainMostDetailed(terrainProvider, allPositions)

// 阶段 3：逐射线判定
for (const { angle, startIdx, count } of rayMeta) {
  const targetH = sampled[startIdx + count - 1]?.height || 0
  let blocked = false
  for (let s = 0; s < count; s++) {
    const terrainH = sampled[startIdx + s]?.height || 0
    const losH = obsHeight + (s+1)/count * (targetH - obsHeight)
    if (terrainH > losH + 0.5) { blocked = true; break }  // 0.5m 容差
  }
  results.push({ angle, distance: blocked ? blockDist : radius, visible: !blocked })
}
```

## 六、缓冲区分析 — GIS 标准算法

### 6.1 点缓冲

72 段正圆近似，纬度方向**不需要**修正（度→米换算固定），经度方向**需要修正**（随纬度变化）：

```js
const cosLat = Math.cos(toRadians(lat))
for (let i = 0; i <= 72; i++) {
  const a = (i / 72) * Math.PI * 2
  coords.push(lon + (radius * Math.cos(a)) / (111_320 * cosLat))  // 经度：需 cos 修正
  coords.push(lat + (radius * Math.sin(a)) / 111_320)              // 纬度：固定换算
}
```

### 6.2 线缓冲

标准 GIS 线缓冲流程：

```
原始折线
  ├── 每个顶点的行进方向角（前后段平均值）
  ├── 左/右垂直方向偏移 → left[] / right[]
  ├── 起点半圆圆帽（连接 left[0] → right[0]）
  └── 终点半圆圆帽（连接 right[last] → left[last]）
        ↓
  多边形环 = 起点帽 + 左侧 + 终点帽 + 右侧逆序
        ↓
  贴地多边形 (ClassificationType.TERRAIN)
```

方向角使用 `atan2` 计算，前后段方向角的**夹角归一化**到 [-π, π]：

```js
let da = angle2 - angle1
if (da > Math.PI)  da -= 2 * Math.PI
if (da < -Math.PI) da += 2 * Math.PI
angle = angle1 + da / 2   // 平分角方向（斜接）
```

## 八、道路名联想搜索 — 三阶段策略

### 8.1 为什么需要自定义联想

高德 Input Tips API 主要返回 POI（Point of Interest），对**道路名**的召回率偏低。为提升用户输入道路名的体验，采用三阶段补全策略。

### 8.2 三阶段流程

```
阶段 1：Input Tips API（快速，主流场景）
  https://restapi.amap.com/v3/assistant/inputtips?keywords=中关村
    → 可能返回少量道路名

阶段 2：Place Search Text API（道路名匹配更好）
  https://restapi.amap.com/v3/place/text?keywords=中关村&city=北京
    → 补充更多道路结果

阶段 3：后缀并行搜索（关键词不含道路后缀时触发）
  并行搜索：中关村大街 | 中关村大道 | 中关村路 | 中关村街 | 中关村高速 | 中关村环路
    → 6 个 Promise.allSettled 并行，聚合结果
```

### 8.3 置信度分级

## 十、性能优化策略

### 10.1 Entity 管理

- **精确清理**：每个视图维护自己的 `entities[]` 或 `trackedEntities[]` 数组，不调用 `viewer.entities.removeAll()`（会误删其他视图的 Entity）
- **按需销毁**：`onUnmounted` 中只清理本视图创建的 Entity

### 10.2 地形采样

- **可视域**：3600 个采样点合并为 **1 次** `sampleTerrainMostDetailed` 调用
- **高度可视化**：首次采样结果缓存到 `cachedDistrictHeights` 对象，二次分析直接复用
- **视线分析**：地形采样失败时降级为椭球面（`height=0`）

### 10.3 交通态势

- **5 分钟缓存**：相同 bbox 的查询结果缓存，避免重复请求
- **4 路并发限制**：避免浏览器并发上限
- **指数退避重试**：1s / 2s，4xx 错误不重试

### 10.4 资源预分配

```js
// 预分配可复用对象，减少 GC（垃圾回收）压力
const _scratchHPR = new Cesium.HeadingPitchRange()
```
