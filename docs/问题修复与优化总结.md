## 一、UI 按钮优化

### 问题 1：按钮尺寸不一致
部分按钮缺少 `size="small"`，默认 32px 与 small 24px 混用。

**修复**：
- `TrafficVisualization.vue`：3 个按钮补 `size="small"`
- `RoutePlanning.vue`："计算路径" 补 `size="small"`
- `SightlineAnalysis.vue`："开始分析" 补 `size="small"`
- `main.css`：`.glass-panel .el-button { font-size: 12px }` 全局兜底

### 问题 2：按钮宽度不一致（并排）
`el-button-group` 中 `flex:1` 无法等分——按钮的最小内容宽度（min-content）阻止了等分。

**修复** (`main.css`)：
```css
.glass-panel .el-button-group .el-button {
  width: auto; min-width: 0;
  overflow: hidden; text-overflow: ellipsis; white-space: nowrap;
}
```
`min-width: 0` 消除最小宽度约束，`flex:1` 真正等分。

### 问题 3：纵向堆叠按钮宽度不一
`el-button` 默认 `display: inline-flex`，不撑满父容器。

**修复** (`main.css`)：
```css
.glass-panel .panel-body .el-button { display: flex; width: 100%; margin: 0; }
.glass-panel .el-button-group .el-button { display: inline-flex; width: auto; }
```
独立按钮改 `display: flex`（块级），button-group 保持 `inline-flex`。

### 问题 4：面板折叠功能

**实现**：
- CSS：`.glass-panel.collapsed .panel-body { display: none }` + 折叠按钮样式
- 每个视图新增 `panelCollapsed` ref + header 内折叠按钮
- 6 个视图统一模式

## 二、视角切换优化

### 问题 1：硬编码北京坐标
点击视角切换始终飞到北京 (116.391, 39.914)，打断数据观察。

**修复**：改用 `camera.positionCartographic` 动态读取当前位置。

### 问题 2：视角模式设计缺陷
- "自由" (−30°) 与 "斜视" (−45°) 仅差 15°，功能重叠
- 名字不反映实际行为

**修复**：三档差异化——俯视(−90°) / 斜视(−45°) / 平视(−10°)，覆盖完整视角范围。

### 问题 3：注视点偏移
相机放在注视点正上方，非垂直视角时视线落在地面远处。

**修复**：ENU 坐标系计算相机偏移 + `Cesium.Transforms.eastNorthUpToFixedFrame()` 精确变换。

### 问题 4：连续切换漂移
每次 `globe.pick()` 的浮点误差累积导致注视区域逐步偏移。

**修复**：`viewTarget` 锁定机制——首次拾取后复用，仅用户拖拽 >50m 时更新。

### 问题 5：复位按钮
**修复**：`resetView()` 改用 Cesium 内置 `camera.flyHome(2.0)`。

### 问题 6：FOV 命名混淆
"视域角度" 与 "可视域分析" 功能名称冲突。

**修复**：改名 "视场角度" + 添加范围标签 + 默认值重置按钮。

## 三、GeoJSON 数据加载与应用

### 问题 1：GeoJSON BOM 头
文件以 UTF-8 BOM (0xFEFF) 开头，不符合 JSON 规范。

**修复**：移除 BOM 字节。

### 问题 2：三次重复加载
`loadGeoJson` / `toggleColorRender` / `toggleHeightVisual` 各自加载同一文件。

**修复**：`ensureGeoJsonLoaded()` 共享加载，`cachedGeoJsonDS` 缓存。

### 问题 3：基础图层被影像遮挡
GeoJsonDataSource 创建的 entity 无 `classificationType`，渲染在影像之下。

**修复**：datasource entity 设为 `show=false`，用 `addBaseOverlay()` 创建 TERRAIN 分类覆盖层。

### 问题 4：分色渲染被遮挡
**修复**：分色 entity 创建时带 `classificationType: Cesium.ClassificationType.TERRAIN`。

### 问题 5：注记位置不准
用包围球中心代替区中心，不规则形状时注记偏移。

**修复**：用 GeoJSON `properties.center`（区政府坐标）。

### 问题 6：颜色随机不稳定
每次 toggle 产生不同颜色。

**修复**：`stableColor(adcode)` — 13 色定性色板 + adcode 哈希取色。

### 问题 7：无法定位（zoomTo 失效）
entity `show=false` 时 `v.zoomTo(ds)` 无法计算范围。

**修复**：将 `show=false` 移到 `await v.zoomTo(ds)` 之后。

### 问题 8：基础图层加载/卸载
**优化**：按钮切换模式（加载↔移除），卸载时自动清除分色和高度可视化。

### 问题 9：分色渲染图例
**优化**：面板内显示色卡对照。

## 四、高度可视化

### 问题 1：Cesium 地形 rectanges 崩溃
`CesiumTerrainProvider` 对中国区 terrain tile 覆盖不完整，availability 四叉树缺少节点。

**修复**：检测到 CesiumTerrainProvider 时提示切换到 ArcGIS 地形。

### 问题 2：ArcGIS 地形 rectanges 崩溃
与 Cesium 同样位置崩溃。

**修复**：不再运行时替换 viewer terrain，改用独立 provider 采样。

### 问题 3：NaN 高度导致崩溃
地形采样返回 NaN → `entity.polygon.height = NaN` → 几何计算异常。

**修复**：过滤 NaN 高度，全部异常时跳过该区。

### 问题 4：采样极慢（ArcGIS 美国服务器）
每次 `sampleTerrainMostDetailed` 从中国访问美国 ArcGIS 服务器 → 1min+。

**修复**：
- 合并采样点：所有区顶点合并为一次 `sampleTerrainMostDetailed` 调用（13 次 → 1 次）
- 顶点降采样：每区最多 10 个采样点
- 高速缓存：Cesium 地形用 `globe.getHeight()` 读 viewer 缓存（瞬时）
- 结果缓存：`cachedDistrictHeights` 存储首次结果，后续调用 <100ms
- 缓存失效：切换地形时自动清除

### 问题 5：采样点重复（MultiPolygon）
多岛区（汉南 5 岛、蔡甸 5 岛）每个环独立采样，同名标签重复。

**未修复**：记录为已知问题。

### 问题 6：toggle off 逻辑
关闭高度/分色渲染时，即使未加载基础图层也会添加 baseOverlay。

**修复**：`if (geoJsonLoaded.value) addBaseOverlay()`。

## 七、高度可视化地形策略（最终版）

```
Cesium 地形 (CDN 快):
  ① globe.getHeight() 读 viewer 缓存
  ② 命中率 ≥ 80% → 瞬时完成

ArcGIS 地形 (美国服务器慢):
  ① globe.getHeight() 读 viewer 缓存
  ② 命中率 < 80% → cachedSampleProvider(独立实例) → sampleTerrainMostDetailed
  ③ 首次 ~5-8s，结果存入 cachedDistrictHeights
  ④ 后续调用 <100ms

切换地形 → cachedDistrictHeights = null → 重新采样
```
