# 视角切换功能优化

## 涉及文件

`src/views/Home.vue` — `switchView()` 函数 + `resetView()` 函数 + 模板视角切换按钮组

---

## 迭代过程

### 问题 1：硬编码坐标（已修复）

**现象**：切换视角时始终飞向北京 (116.391, 39.914)，无论用户浏览到哪个区域。

**根因**：`destination` 写死为 `[116.391, 39.914, height]`。

**修复**：改用 `camera.positionCartographic` 动态读取当前位置。

---

### 问题 2：视角模式设计缺陷（已修复）

**现象**：
- "自由"视角名不副实，实际只是 `pitch=-30°` 的固定预设
- 与"斜视"(`pitch=-45°`) 仅差 15°，功能重叠
- 缺少真正差异化的视角（如近水平地面视角）

**修复**：三档差异化，覆盖完整视角范围：

| 模式 | pitch | 间隔 | 视觉定位 |
|------|-------|------|---------|
| 俯视 | -90° | — | 正上方地图视角 |
| 斜视 | -45° | 45° | 3D 地形俯瞰 |
| 平视 | -10° | 35° | 近水平天际线视角 |

按钮由"自由"重命名为"平视"，key 由 `'free'` 改为 `'shallow'`。新增 `currentView` ref 实现按钮激活态高亮。

---

### 问题 3：注视点偏移 — 相机位置未适配 pitch 角（已修复）

**现象**：非垂直视角下，相机放在注视点正上方但斜向看去，视线落在地面远处而非目标区域。下图说明：

```
之前（错误）:
  相机在目标正上方高度 h
  pitch=-45° → 视线斜向远方，看到的地面位置完全不同于目标

  目标 ●──── 相机 ✈︎ (正上方)
              ╲
               ╲ 视线方向 (-45°)
                ╲
    ─────────────●──────────────── 地面
                 实际看到的区域（远离目标）
```

**根因**：`destination` 直接设为 `(targetLon, targetLat, distance)`，即目标正上方。对 `pitch=-90°` 正确，但对 `-45°` 和 `-10°` 错误。

**修复**：在注视点的 ENU 局部坐标系中计算相机偏移。相机应位于注视点的反方向（anti-heading），高度和水平距离由 pitch 决定：

```
正确做法:
  h = d × sin(|pitch|)     相机相对注视点的高度
  r = d × cos(|pitch|)     水平偏移距离
  相机在注视点反方向 r 米，高度 h 米

  俯视(-90°): h=d,  r=0      → 正上方
  斜视(-45°): h=0.71d, r=0.71d → 侧上方 45°
  平视(-10°): h=0.17d, r=0.98d → 近乎水平远眺

  相机 ✈︎ (反方向，r 米外，高 h 米)
        ╲
         ╲ 视线方向 (pitch)
          ╲
  ─────────●────────────────── 地面
           目标 ✓
```

关键代码：
```js
// ENU 偏移：相机在注视点的反方向
const enuOffset = new Cesium.Cartesian3(
  -distance * Math.cos(pitch) * Math.sin(heading), // East
  -distance * Math.cos(pitch) * Math.cos(heading), // North
  -distance * Math.sin(pitch)                       // Up
)

// ENU → ECEF 精确变换（Cesium 原生，WGS84 椭球面）
const transform = Cesium.Transforms.eastNorthUpToFixedFrame(viewTarget)
const cameraPos = Cesium.Matrix4.multiplyByPoint(transform, enuOffset, new Cesium.Cartesian3())
```

### 问题 4：连续切换注视点漂移 — globe.pick 浮点误差（已修复）

**现象**：连续点击不同视角按钮时，注视区域逐步漂移。

**根因**：每次调用 `switchView()` 都重新执行 `globe.pick()` 拾取注视点。不同相机角度下虽然看着同一区域，但拾取结果存在微小浮点差异（< 1m），连续切换几次后累积成明显偏移。

**修复**：引入 `viewTarget` 锁定机制：

```js
let viewTarget = null // 锁定的注视点（Cartesian3）

// 首次拾取后锁定，后续切换复用
// 仅在用户手动拖拽超过 50m 时才更新
if (lookAt) {
  if (!viewTarget || Cesium.Cartesian3.distance(lookAt, viewTarget) > 50) {
    viewTarget = lookAt.clone()
  }
}
```

```
切换流程：
  俯视 → 斜视 → 平视 → 俯视 → ...
    │       │       │       │
    └───────┴───────┴───────┘
        所有切换共享同一个 viewTarget
        注视点完全锁定，零漂移

用户手动拖拽 > 50m → viewTarget 更新 → 重新锁定
```

---

### 问题 5：偏移计算使用平面近似导致漂移（已修复）

**现象**：即使锁定了 viewTarget，计算出的相机位置在椭球面上仍有偏差。

**根因**：第一版偏移计算使用了平面地球近似：

```js
// 错误：球面距离用平面近似
const dLon = (r * Math.sin(antiHeading)) / (111320 * Math.cos(latRad))
const dLat = (r * Math.cos(antiHeading)) / 111320
```

`111320 * cos(lat)` 只在地球为完美球体且在小范围内成立。WGS84 椭球面上，10km+ 距离就可产生米级偏差。

**修复**：使用 Cesium 原生 `Cesium.Transforms.eastNorthUpToFixedFrame()` 进行 ENU → ECEF 变换。该函数基于 WGS84 椭球参数，数学上精确，避免任何平面近似。

```js
// 正确：Cesium 原生椭球面变换
const transform = Cesium.Transforms.eastNorthUpToFixedFrame(viewTarget)
const cameraPos = Cesium.Matrix4.multiplyByPoint(transform, enuOffset, new Cesium.Cartesian3())
```

## 最终架构

```
switchView(type)
  │
  ├─ ① globe.pick(screenCenter) → lookAt 拾取当前注视点
  │
  ├─ ② viewTarget 锁定判断
  │     if (无锁定 || 用户拖拽 > 50m) → 更新锁定
  │     else → 复用锁定
  │
  ├─ ③ distance = clamp(dist(camera, viewTarget), 500, 50000)
  │
  ├─ ④ views[type] → { pitch: -90°|-45°|-10°, heading: camera.heading }
  │
  ├─ ⑤ ENU 偏移计算
  │     enuOffset = (-d·cosP·sinH, -d·cosP·cosH, -d·sinP)
  │
  ├─ ⑥ Cesium.Transforms.eastNorthUpToFixedFrame(viewTarget)
  │     → ECEF 精确变换
  │
  └─ camera.flyTo({ destination, orientation, duration: 1.2 })
```

## 辅助功能

**复位全局视角** (`resetView`)：调用 Cesium 内置 `camera.flyHome(2.0)` 回到 Viewer 初始化默认视角，替代早期硬编码的北京坐标复位。

**激活态指示**：`currentView` ref 追踪当前模式，按钮动态绑定 `:type`（`'primary'` / `'default'`），用户可见当前激活的视角。

## 验证

1. `npm run dev` → 基础三维场景
2. 加载 3D Tiles 或 GeoJSON，浏览到任意区域
3. 连续点击 俯视/斜视/平视 → 注视区域应始终稳定不偏移
4. 手动拖拽地图到新位置，再次点击视角按钮 → 应适配新的注视区域
5. 点击复位 → 回到 Cesium 默认初始视角
